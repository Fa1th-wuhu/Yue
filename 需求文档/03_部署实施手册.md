# “简阅”小说阅读平台——系统部署实施手册

---

## 一、 部署前准备与环境要求
本部署指南指导在信创标准基础环境（鲲鹏 ARM64 + openEuler 操作系统）下一键容器化发布。

### 1.1 基础环境依赖清单
| 环境项 | 推荐规格 | 作用说明 |
| :--- | :--- | :--- |
| **CPU / 内存** | 2vCPU / 4GB 内存或以上 | 保障 FastAPI 异步应用与容器系统开销正常 |
| **操作系统** | openEuler 22.03 LTS SP2 | 安全自主可控的底层 Linux 操作系统环境 |
| **系统盘空间** | 可用物理空间 > 15GB (总空间 40GB) | 用于存放容器基础镜像、项目源码及数据卷备份 |
| **核心运行时** | Docker CE 24.x / Compose v2.20+ | 提供轻量级应用封装，屏蔽底层处理器架构细节 |
| **打包环境依赖** | unzip / zip 工具 | 用于解压缩部署包 |
| **网络要求** | 开放对外 TCP 8000 端口 | 允许外界客户端浏览器直接访问 Web 服务API |

## 二、 一键部署实施步骤
本实施方案采用完全自动化的一键部署设计。在运维人员手动通过安全协议 (SCP) 将代码压缩包 `Yue.zip` 上传至目标宿主机服务器的 `/root/` 下后，即可按以下动作实施：

### 步骤 1： 登录服务器并获取管理员权限
使用终端工具（如 SecureCRT / Termius）通过 SSH 安全协议登录到 openEuler 虚拟机的命令行界面，并切换到 `root` 用户。

### 步骤 2： 确认部署源码包的存在性
运行以下命令检查 `/root/Yue.zip` 文件是否存在于宿主机磁盘：
```bash
ls -la /root/Yue.zip
# 正确输出：-rw-r--r-- 1 root root 512034 Jun 4 15:00 /root/Yue.zip
```

### 步骤 3： 启动一键部署实施脚本
若是系统首次部署或重新发布，可以直接解压并运行项目自带的一键编译与构建脚本：
```bash
# 1. 解压至指定目录
unzip -o /root/Yue.zip -d /root/Yue

# 2. 切换至运维脚本路径并附加执行权限
cd /root/Yue/ops
chmod +x deploy_yue.sh

# 3. 运行部署脚本
./deploy_yue.sh
```

### 步骤 4： 核心自动化部署逻辑解析
运行 `deploy_yue.sh` 后，脚本内部会自动推进如下部署保障：
1. **优雅下线旧服务**：若系统此前在运行，自动进入 `/root/Yue` 调用 `docker-compose down` 优雅断开、停止正在运行的旧容器，并释放被 SQLite 独占的文件句柄，防止数据锁冲突。
2. **兼容性热修补**：自动读取 `docker-compose.yml`。如果发现头部由于软件版本兼容丢失 `version: "3.7"`，自动利用 `sed` 将规范行写入文件头。
3. **动态 IP 校准**：自动读取并修改 `code_artifact.html` 中的前端 AJAX 访问端点，将开发环境写死的 `127.0.0.1:8000` 动态编译替换为真实部署机 IP 地址（例如 `10.42.78.219:8000`），实现一键部署零手动配置。
4. **一键容器化重构**：执行 `docker-compose up -d --build` 重新拉取 python 底座，重新封装打包整个 backend 代码，并自动拉起，对持久化数据库卷进行保护性挂载。
5. **服务自检**：调用本地服务健康度：`curl http://127.0.0.1:8000/health`，若响应 200，则在屏幕打出“部署完成”。

## 三、 系统访问与功能自测

### 3.1 客户端访问方式
- **主页网页访问**：在客户端浏览器直接键入地址 `http://10.42.78.219:8000/`，即可显示小说主页。
- **API 交互式文档**：在浏览器键入 `http://10.42.78.219:8000/docs` 访问系统自动生成的 Swagger UI 调试大厅。
- **健康度查询**：在浏览器键入 `http://10.42.78.219:8000/health` 查看系统实时负载状态。

### 3.2 系统内置基础测试账号
在首次部署后，启动即内置了测试数据，可以直接使用以下凭证登录和操作：
- **系统管理员**：账号 `admin` 密码 `123456`（拥有发布小说、增删分类权限）。
- **普通测试用户**：账号 `user1` 密码 `123456`（拥有阅读、收藏到书架、点赞、发表评论权限）。

### 3.3 标准功能演示推进流程 (3分钟快速汇报/验收路径)
1. **登录验证**：使用普通账号 `user1` 进行登录，验证系统状态是否由“未登录”状态正确转换为显示用户的登录令牌信息。
2. **分类检索**：点击分类导航（如“科幻”、“武侠”），小说卡片区实时局部刷新。验证分类过滤机制无误。
3. **小说阅读（重点）**：点击《三体·地球往事》阅读按钮，进入幽灵倒计时和科学边界阅读。验证章节列表与章节内容渲染正常。
4. **点赞与收藏**：点击一键“点赞”并点击“加入书架”按钮。刷新页面或切换至“书架”选项卡，验证小说已被永久保存在个人私人书架上。
5. **评论交互**：在小说详情页的评论输入框中打入评论内容“宇宙终结，农场主在感恩节进来了！”，并点击发送，在下方评论展示区实时出现评论记录。验证闭环打通。
