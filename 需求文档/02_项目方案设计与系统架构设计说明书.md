# “简阅”小说阅读平台——项目方案设计与系统架构设计说明书

---

## 一、 系统概述
“简阅” (Yue) 是针对信创（信息技术应用创新）底座深度定制的、高安全可信的轻量级小说在线阅读平台。系统采用前后端分离架构，前端以轻量、极简单网页的形式呈现，后端依托 Python 高性能异步 Web 框架 FastAPI 提供稳定、高并发的 Restful API 支撑。整个系统完全支持在国产芯片（鲲鹏 ARM64）、国产虚拟化（FusionCompute 8.x）及国产操作系统（openEuler）构成的“信创基线”上无缝运行，满足国家对核心关键应用安全自主可控的要求。

---

## 三、 系统架构设计

### 3.1 总体拓扑结构
系统的物理部署拓扑采用三层纵深防御体系：
1. **外网网关防护层**：用户浏览器发起请求（HTTP 80 或 HTTPS 443）首先到达虚拟机的 Nginx 网关。Nginx 负责 SSL 证书卸载、静态资源托管，并将动态 API 请求反向代理至后端。
2. **容器应用层**：由 Docker Compose 隔离出来的专有容器网络内部运行 `yue-app`。该容器搭载 Uvicorn 高性能 ASGI 服务器，运行 FastAPI 应用，处理路由分发与业务逻辑。
3. **数据持久层**：采用 Docker 命名数据卷 `yue_data` 挂载宿主机，对嵌入式 SQLite 3 数据库文件 `yue.db` 进行热锁定与持久化。保证数据不随容器销毁而丢失。

```text

              │ (HTTPS - 443 / HTTP - 80)
              ▼
   ┌───────────────────────────────────────────────┐
   │          鲲鹏虚拟机 (openEuler OS)             │
   │                                               │
   │   ┌───────────────────────────────────────┐   │
   │   │             Nginx 1.24.x              │   │
   │   │  (SSL证书卸载、静态转发、API请求代理)   │   │
   │   └──────────────────┬────────────────────┘   │
   │                      │ (内部转发: 8000)       │
   │                      ▼                        │
   │   ┌───────────────────────────────────────┐   │

   │   │                                       │   │
   │   │   ┌───────────────────────────────┐   │   │
   │   │   │        yue-app 容器           │   │   │
   │   │   │  (Uvicorn + FastAPI 后端)     │   │   │

   └───────────────────────────────────────────────┘
```

### 3.2 逻辑架构设计
系统基于经典的 Model-View-Controller (MVC) 演进的 API 逻辑架构：
- **视图层(View)**：单页面 HTML/JS (`code_artifact.html`)，采用异步 AJAX 与后端交互。
- **控制层(Controller/Route)**：在 `backend/app/main.py` 中，定义 Restful 路由接收前端参数，进行登录会话校验、数据实体拦截，并调用 ORM 操作。
- **业务数据层(Model)**：在 `backend/app/models.py` 中基于 SQLAlchemy 定义六大数据库映射表，并支持外键级联删除。

## 四、 数据库表结构设计

### 4.1 用户表 (users)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 用户自增唯一ID |
| `username` | `String(64)` | UNIQUE, NOT NULL, INDEX | 用户名（登录凭证，支持快速检索） |
| `password` | `String(128)` | NOT NULL | 用户密码（哈希存储） |
| `created_at` | `DateTime` | DEFAULT (now()) | 账号注册生成时间 |

### 4.2 分类表 (categories)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 分类自增唯一ID |
| `name` | `String(64)` | UNIQUE, NOT NULL, INDEX | 小说分类名称（科幻、武侠等） |

### 4.3 图书表 (books)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 小说自增唯一ID |
| `title` | `String(128)` | NOT NULL | 小说标题名称 |
| `author` | `String(128)` | NOT NULL | 小说作者姓名 |
| `category` | `String(64)` | NOT NULL, INDEX | 关联分类名称（支持跨表聚合） |
| `views` | `Integer` | NOT NULL, DEFAULT 0 | 浏览阅读量 |
| `likes` | `Integer` | NOT NULL, DEFAULT 0 | 点赞量统计 |
| `is_daily_rec` | `Boolean` | NOT NULL, DEFAULT False | 是否为每日推荐小说 |
| `cover_img` | `String(512)` | NOT NULL, DEFAULT '' | 小说封面图/缩略图路径 |

### 4.4 章节表 (chapters)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 章节自增唯一ID |
| `book_id` | `Integer` | FOREIGN KEY (books.id) ON DELETE CASCADE | 归属的小说ID（外键级联） |
| `chapter_num` | `Integer` | NOT NULL | 章节序号（第几章） |
| `title` | `String(128)` | NOT NULL | 章节标题 |
| `content` | `Text` | NOT NULL | 章节核心文本内容 |

> **约束说明**：在 `[book_id, chapter_num]` 字段组合上建立了唯一性索引（uq_book_chapter_num），确保单本小说的章节序号绝不重复。

### 4.5 书架关联表 (bookshelf)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 书架自增唯一ID |
| `user_id` | `Integer` | FOREIGN KEY (users.id) ON DELETE CASCADE | 归属的用户ID（外键级联） |
| `book_id` | `Integer` | FOREIGN KEY (books.id) ON DELETE CASCADE | 收藏的小说ID（外键级联） |

> **约束说明**：在 `[user_id, book_id]` 上建立唯一联合约束（uq_user_book），防止同一用户将同一本书重复加入书架。

### 4.6 评论表 (comments)
| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `Integer` | PRIMARY KEY, AUTO_INCREMENT | 评论自增唯一ID |
| `user_id` | `Integer` | FOREIGN KEY (users.id) ON DELETE CASCADE | 发布评论的用户ID |
| `username` | `String(64)` | NOT NULL | 发布时的用户冗余用户名 |
| `book_id` | `Integer` | FOREIGN KEY (books.id) ON DELETE CASCADE | 被评论的小说ID |
| `content` | `Text` | NOT NULL | 评论详情内容 |
| `created_at` | `String(32)` | NOT NULL | 格式化好的时间戳（如 2026-06-04 15:30） |

## 五、 核心业务接口 (API) 设计
简阅后端提供了一套纯净的 JSON Restful API 规范，用于客户端异步联调：
* **会话管理**：
  * `POST /api/users/login` (用户登录校验)
  * `POST /api/users/register` (用户注册)
* **图书检索**：
  * `GET /api/books` (图书列表)
  * `POST /api/books` (发布新书)
  * `PUT /api/books/{book_id}` (修改分类与信息)
  * `DELETE /api/books/{book_id}` (级联删除小说)
  * `POST /api/books/{book_id}/like` (小说点赞累加)
* **章节管理**：
  * `GET /api/chapters` (章节列表)
  * `POST /api/chapters` (创建新章节)
  * `PUT /api/chapters/{chapter_id}` (修改章节详情)
* **个人书架**：
  * `GET /api/bookshelf` (个人收藏夹检索)
  * `POST /api/bookshelf` (加入书架，防止重复插入)
  * `DELETE /api/bookshelf/{item_id}` (从书架移出)
* **评论大厅**：
  * `GET /api/comments` (获取大厅所有评论列表)
  * `POST /api/comments` (发布新评论，即时刷新)
* **监控服务**：
  * `GET /health` (健康监控自愈探针)
