# “简阅”小说阅读系统——运行环境（信创基线）设计与部署说明书

---

## 1. 概述

### 1.1 项目简介
**“简阅” (Yue)** 是一款轻量级、响应式的前后端分离小说阅读平台。系统采用极简的单页面前端网页（`code_artifact.html`）结合基于 FastAPI 框架的 Python 异步高性能后端（`backend/app`），为用户提供小说书架、分类浏览、章节阅读、点赞和实时评论等核心功能。

### 1.2 信创建设背景
信创（信息技术应用创新）是我国信息化建设的国家战略。本说明书旨在针对“简阅”系统的生产部署，设计一套**符合国产化自主可信要求（信创基线）**的系统运行环境与容器化部署方案。通过依托国产鲲鹏硬件算力底座、openEuler 国产操作系统、华为 FusionCompute 虚拟化管理，结合业界成熟的 Docker 容器引擎与 Nginx 反向代理，确保“简阅”系统的高安全、高可用、可控制及快速迁移能力。

## 3. 部署架构设计

“简阅”系统采用轻量化、容器隔离的部署架构。整体拓扑如下图所示：

```text
       [ 终端用户浏览器 ]
              │ (HTTPS - 443 / HTTP - 80)
              ▼
   ┌───────────────────────────────────────────────┐
   │          鲲鹏虚拟机 (openEuler OS)             │
   │                                               │
   │   ┌───────────────────────────────────────┐   │
   │   │             Nginx 1.24.x              │   │
   │   │  (SSL证书卸载、静态转发、API请求代理)   │   │
   │   └──────────────────┬────────────────────┘   │
   │                      │ (内部转发: 8000)       │
   │                      ▼                        │
   │   ┌───────────────────────────────────────┐   │
   │   │           Docker 容器网络             │   │
   │   │                                       │   │
   │   │   ┌───────────────────────────────┐   │   │
   │   │   │        yue-app 容器           │   │   │
   │   │   │  (Uvicorn + FastAPI 后端)     │   │   │
   │   │   └──────────────┬────────────────┘   │   │
   │   │                  │ (SQLAlchemy ORM)   │   │
   │   │                  ▼                    │   │
   │   │         ┌──────────────────┐          │   │
   │   │         │ yue_data (数据卷)│          │   │
   │   │         │ ├─ yue.db (SQLite│          │   │
   │   │         └──────────────────┘          │   │
   │   └───────────────────────────────────────┘   │
   └───────────────────────────────────────────────┘
```

### 3.1 部署核心设计：
1. **静态与动态整合**：由于项目前端 `code_artifact.html` 体积小、交付便利，已打包进入 `yue-app` 容器中。后端 FastAPI 根路径 `/` 直接通过 `FileResponse` 返回该 HTML 文件，降低了静态资源部署的复杂度。
2. **零依赖启动**：后端基于 `SQLAlchemy` 实现了自动化表结构创建（`Base.metadata.create_all`）和基础数据播种（`seed.seed_data`），使容器在一键拉起后即可直接可用。
3. **数据卷持久化**：数据库文件存储在外部映射的 `yue_data` 数据卷中（对应主机 `/var/lib/docker/volumes/yue_data/_data` ），即使应用容器销毁或重建，业务数据依然完好无损。

## 4. 部署与实施方案

根据项目在 `ops/` 目录下提供的自动化运维脚本，可以快速在 openEuler 虚拟机上完成一键部署。

### 4.1 部署包结构准备
在进行部署前，需将代码打包为 `Yue.zip`：
* 包含 `backend/` 目录（requirements.txt 依赖及 app 源码）
* 包含 `Dockerfile` 与 `docker-compose.yml`
* 包含前端页面 `code_artifact.html`
* 包含初始化脚本 `add_santi.py`（如有需要手动扩充书库）

### 4.2 部署执行步骤 (`deploy_yue.sh`)
系统一键部署脚本已在项目 `@/ops/deploy_yue.sh:1-35` 中定义。在鲲鹏云主机上执行以下步骤：

1. **上传部署包**：将 `Yue.zip` 上传至虚拟机的 `/root/Yue.zip` 路径下。
2. **执行自动化部署**：
   ```bash
   chmod +x /root/Yue/ops/deploy_yue.sh
   /root/Yue/ops/deploy_yue.sh
   ```
3. **部署脚本逻辑解析**：
   * 自动停止旧的容器实例：`docker-compose down || true`。
   * 清理并重新解压最新部署包。
   * **自动匹配 IP 地址**：使用 `sed` 将前端的开发测试接口地址 `http://127.0.0.1:8000/api` 替换为局域网/云主机真实可达 IP （如信创环境内部 IP `http://10.42.78.219:8000/api`）。
   * **一键容器化构建与拉起**：运行 `docker-compose up -d --build`，使用多阶段或轻量级 `python:3.12-slim` 基础镜像进行构建，确保镜像体积小，提高安全性。
   * **自检测试**：通过对 `/health` 接口发起健康检查请求（`curl`），并在终端输出容器实时运行状态。

## 5. 自动化运维与容灾备份方案

信创基线不仅对硬件和基础软件有要求，对系统运行的**稳定性监控**和**数据安全性**也提出了极高标准。

### 5.1 自动化巡检与监控机制 (`monitor_yue.sh`)
运行状态监控由部署于 openEuler 虚拟机的 `@/ops/monitor_yue.sh:1-24` 脚本定时触发（可写入 crontab 中每小时或每日执行）：

1. **容器状态巡检**：利用 `docker ps --filter "name=yue-app"` 检查应用容器的存活状态（Up/Exited）。
2. **资源占用监控**：利用 `docker stats --no-stream` 采集单次 CPU、内存、网络 I/O 及磁盘 I/O 数据，防止应用由于内存泄露拖垮宿主机。
3. **健康检查检测**：利用 `curl` 定时请求内网 API 的健康状态端点 `http://127.0.0.1:8000/health`。
4. **日志异动排查**：定时拦截最新的 50 行应用日志（`docker logs --tail 50`）输出，并转储或上报。
5. **磁盘空间预警**：调用系统级 `df -h` 监控虚拟机系统盘（40GB）的使用率，防止因日志未做轮转等原因导致存储满额挂死。

### 5.2 自动备份与容灾机制 (`backup_yue.sh`)
业务数据的高安全冷备份由 `@/ops/backup_yue.sh:1-32` 脚本实现：

1. **双向备份机制**：
   * **数据备份**：通过 `docker cp` 直接将正在运行的容器中的 SQLite 数据库文件 `/app/data/yue.db` 安全复制到宿主机的备选分区 `/root/backups/yue/yue_YYYYMMDD_HHMMSS.db`。
   * **代码与配置备份**：将 `code_artifact.html`、`docker-compose.yml`、`Dockerfile`、整个 `backend/` 及初始化脚本打包为 `yue_project_YYYYMMDD_HHMMSS.tar.gz` 格式归档。
2. **滚动自动清理**：脚本中使用 `find` 指令自动查找并删除 `7天前`（`+7`天）的所有旧备份，保证虚拟机 40GB 系统盘的空间利用率和良性循环。

### 5.3 故障数据一键恢复 (`restore_yue_db.sh`)
当发生物理故障或数据库损坏时，可通过 `@/ops/restore_yue_db.sh:1-24` 脚本，配合历史备份一键重构数据库：

1. **调用规范**：
   ```bash
   ./restore_yue_db.sh /root/backups/yue/yue_YYYYMMDD_HHMMSS.db
   ```
2. **恢复机制**：
   * 自动停止当前的 Docker 服务环境，防止由于写锁竞争导致数据恢复损坏：`docker-compose down`。
   * 重新拉起应用基础环境。
   * 使用 `docker cp` 将安全的数据库备份复写至容器的挂载目录中。
   * 重启 `yue-app` 容器，快速恢复正常的读取和读写服务。

## 6. 总结与演进方向

基于“信创基线”建设的“简阅”小说阅读系统，具备了极高的自主可控性与环境适应能力。
* **硬件级高适配**：通过 `python:3.12-slim` 及 Docker 容器抽象，使应用在不更改任何代码的情况下，完美运行在鲲鹏等国产 CPU 架构上。
* **软件级高安全**：采用 openEuler 国产操作系统底层，并实现了严格的定时数据冷备份、自动化巡检和环境自动拉起。
* **灵活的数据演进**：当前基于 SQLite 3 满足单机轻量级信创应用；由于采用 `SQLAlchemy ORM` 统一数据访问层，未来若需要升级为大并发、高可用集群，可随时通过调整 `DATABASE_URL`，将数据库切换到高可用版 **MySQL 8.0.x** 甚至国产 **达梦/GaussDB**，前端与业务逻辑层完全不需重构。
